# Release 0.2.0

## Highlights

This release adds two new entry points to `@vanya2h/utils`: `./models` for domain-oriented Zod schemas (numeric types, viem blockchain types, and branded IDs) and `./zod` for schema utility helpers. The `./common` entry point gains `serializeApi` for deterministic API object serialization.

## Changes

### New Features

- **`./models` entry point** — New module with Zod-based domain models:
  - `IdSchema` — factory for creating branded string ID schemas (`src/models/id.ts`)
  - `Int256`, `Int256Positive`, `Int256Json`, `toInt256Json`, `createInt256` — BigInt/Int256 types with JSON-serializable variants (`src/models/int256.ts`)
  - `NumberishSchema`, `NumberishJsonSchema`, `StringNumberSchema`, `INumberish`, `INumberishJson` — union schemas accepting `string | number | bigint` (`src/models/numberish/numberish.ts`)
  - `AddressSchema`, `IAddressString` — validates EVM addresses via viem (`src/models/viem/address.ts`)
  - `HexStringSchema`, `IHexString` — validates hex strings via viem (`src/models/viem/hex.ts`)
  - `TransactionHashSchema`, `ITransactionHash` — validates transaction hashes via viem (`src/models/viem/transactionHash.ts`)
  - `SupportedChainIds`, `createSupportedChains` — typed chain ID unions and chain lookup helpers (`src/models/viem/supportedChainId/supportedChainId.ts`)

- **`./zod` entry point** — New module with Zod utility helpers:
  - `bySchema` — parse a value with a Zod schema (throws on failure) (`src/zod/bySchema.ts`)
  - `ifSchema` / `IfSchemaEmpty` — safe-parse a value, returning a sentinel on failure (`src/zod/ifSchema.ts`)
  - `isSchema` — type guard using a Zod schema (`src/zod/isSchema.ts`)

- **`serializeApi`** — Added to `./common`; serializes objects to deterministic JSON with sorted keys, compact Zod schema representations, and function stubs (`src/common/serializeApi.ts`)

- **RC Release workflow** — New GitHub Actions workflow (`.github/workflows/rc-release.yml`) that automatically publishes RC packages to npm on push to `staging`

### Breaking Changes

- **`zod` upgraded from v3 to v4** (`^3.24.2` → `^4.3.6`). Consumers using Zod types from this package must also upgrade to Zod v4.
- **`zod` moved from `peerDependencies` to `dependencies`** — no longer needs to be installed separately by consumers.

### Dependencies

- `zod` `^4.3.6` — moved from `peerDependencies`/`devDependencies` to `dependencies`, upgraded from v3 to v4
- `viem` `^2.48.1` — added as a new dependency to support blockchain type validation in `./models`
