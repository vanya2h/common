# Release 0.4.0

## Highlights

This release adds two new entry points to `@vanya2h/utils`: a `models` module with Zod-based primitives for common blockchain types, and a `zod` module with utility helpers for safe schema parsing. It also introduces `serializeApi`, a deterministic JSON serializer for API objects, and a new RC release workflow for automated pre-release publishing.

## Changes

### New Features

- **`models` entry point** (`packages/utils/src/models/`) — new Zod schemas and types for:
  - `NumberishSchema` / `INumberish` — union of `string | number | bigint`; `NumberishJsonSchema` for JSON-safe subset; `StringNumberSchema` for locale-tolerant string-to-number coercion (`numberish/numberish.ts`)
  - `Int256` / `Int256Positive` — bigint schemas built on `NumberishSchema`; `Int256Json` branded string type; `toInt256Json()` conversion helper (`int256.ts`)
  - `IdSchema<T>` — generic branded string schema factory for typed IDs (`id.ts`)
  - `AddressSchema` / `IAddressString` — validates and types viem `Address` strings (`viem/address.ts`)
  - `HexStringSchema` / `IHexString` — validates and types viem `Hex` strings (`viem/hex.ts`)
  - `TransactionHashSchema` / `ITransactionHash` — validates and types viem `Hash` strings (`viem/transactionHash.ts`)
  - `SupportedChainIds()` / `createSupportedChains()` — type-safe chain ID union and helper from a viem `Chain[]` array (`viem/supportedChainId.ts`)

- **`zod` entry point** (`packages/utils/src/zod/`) — schema utility helpers:
  - `bySchema(schema, x)` — parse `x` through `schema`, throwing on failure (`bySchema.ts`)
  - `isSchema(schema, x)` — type-guard returning `true` if `x` satisfies `schema` (`isSchema.ts`)
  - `ifSchema(schema, x)` — safe-parse returning the parsed value or `IfSchemaEmpty` sentinel on failure (`ifSchema.ts`)

- **`serializeApi(obj)`** (`packages/utils/src/common/serializeApi.ts`) — deterministic `JSON.stringify` with alphabetically sorted keys; serializes Zod schemas to compact human-readable strings and functions to `[Function: name]`.

- **RC release workflow** (`.github/workflows/rc-release.yml`) — automated pre-release publishing on push to `staging`: enters changeset pre-release mode, versions packages, pushes bumps, builds, and publishes to npm.

- **Claude slash commands** (`.claude/commands/`) — added `commit.md`, `pr.md`, `pr-update.md`, and `release-docs.md` for standardized repo workflows.

### Dependencies

- `viem` is now a dependency of `@vanya2h/utils` (used by the new `models/viem` schemas).
