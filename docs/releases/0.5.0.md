# Release 0.5.0

## Highlights

Introduces `@vanya2h/prettier-config`, a new shareable Prettier configuration package, and integrates it into `@vanya2h/eslint-config` so formatting options are defined in one place and shared across both tools. All packages in the fixed group bump to `0.5.0`.

## Changes

### New Features

- **`@vanya2h/prettier-config`** — new package exporting a standard Prettier config (`semi: true`, `trailingComma: "all"`, `printWidth: 120`, `tabWidth: 2`, `singleQuote: false`). Can be consumed via `prettier.config.ts`, `package.json#prettier`, or extended with overrides. See `packages/prettier-config/src/index.ts`.

- **`@vanya2h/eslint-config` — Prettier options sourced from shared config** — the `prettier/prettier` ESLint rule now receives options directly from `@vanya2h/prettier-config`, ensuring ESLint and the Prettier formatter always agree. See `packages/eslint-config/src/base.ts`.

### Breaking Changes

- **Import sort group removed** — the `@evergonlabs`-specific import sort group has been removed from `@vanya2h/eslint-config/base`. Imports previously sorted into a dedicated `@evergonlabs/*` bucket will now fall into the generic external package group. Consumers should re-run `eslint --fix` after upgrading to reorder affected imports.

### Dependencies

- `@vanya2h/eslint-config` now has a peer/workspace dependency on `@vanya2h/prettier-config`.
- Node.js engine requirement raised to `>=25` across all packages and CI workflows.
